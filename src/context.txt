Now let's get to know more about PyPore3D and SOME of its functions : 
**PyPore3D** is an open-source Python library for 3D/4D tomographic image processing and analysis of porous/multiphase materials. Developed by SYRMEP collaboration at Elettra Sincrotrone Trieste (Italy).

**Architecture:** Python wrappers (via SWIG) around optimized C core (Pore3D library with OpenMP parallelization). Integrates with SimpleITK, NumPy, SciPy, Plotly.

**Source:** https://gitlab.elettra.eu/aboulhassan.amal/PyPore3D

**Target Users:** Geologists, biologists, materials scientists, pharmacists analyzing CT/microscopy data.

**Platform:** Linux (primary), Windows (tested). Runs on commodity hardware (tested: 8-core, 16GB RAM).

## KEY CONCEPTS

| Term | Definition |
|------|------------|
| **RAW format** | Headerless binary image files. Dimensions must be known beforehand. |
| **8-bit** | Unsigned byte (0-255). Use `bytearray` in Python. |
| **16-bit** | Unsigned short (0-65535). Use `ushort_array` in Python. |
| **Binary image** | Segmented: 0=background, 255 (or 65535)=object. |
| **Connectivity** | Voxel neighborhood: 6=faces, 18=faces+edges, 26=all neighbors. |
| **Resolution** | Physical voxel size in mm (e.g., 0.001 = 1 Âµm). |
| **VOI** | Volume of Interest - subset of full dataset. |
| **Skeleton** | 1-voxel-thick centerline representation of structure. |

### SEGMENTATION

- methodNum: 1=Kittler, 2=Otsu, 3=Pun, 4=Ridler, 5=Kapur, 6=Tsai, 7=Huang-Yager

## KNOWN ISSUES & PITFALLS

1. **Gaussian Filter Bug:** `py_p3dGaussianFilter8/16` ignores width/sigma params. Always uses width=3, sigma=1.0.

2. **Ring Removal 2D Only:** `py_p3dSijbersPostnovRingRemover` works on single 2D slices, not 3D volumes.

3. **Dimension Mismatch:** Wrong dimensions corrupt output silently.

4. **Unsigned Only:** All functions expect unsigned data. Signed data will produce incorrect results.

5. **Memory:** Anisotropic diffusion and GVF skeletonization are memory-intensive.

6. **Kernel Size:** Must be ODD for all filter widths (3, 5, 7, ...).

7. **16-bit File-Based Functions:** Some p3dSITKPy_16 functions use file I/O (input .raw, output .mhd) instead of memory arrays.

## OUTPUT INTERPRETATION

### Euler Characteristic (Cv/Îºv)
- Positive: More isolated objects than connections (disconnected)
- Negative: More connections/tunnels than objects (highly connected)
- Near zero: Balanced network

### Isotropy Index (I)
- 1.0: Perfectly isotropic (no directional preference)
- 0.0: Perfectly anisotropic (strong alignment)
- Typical porous materials: 0.8-0.95

### Sphericity
- 1.0: Perfect sphere
- <0.5: Highly irregular/elongated
- Decreases with increasing volume (larger objects more deformed)
